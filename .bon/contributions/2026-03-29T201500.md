The CLI is transitioning from single-user to team use. The SDK (todoist-api-python v4) doesn't wrap the `/api/v1/user` endpoint, so identity is resolved via a direct httpx call to the same REST v1 base URL. This is a seam — if Todoist changes the user endpoint, the SDK changelog won't flag it. The deeper design question is how to distinguish "personal project with collaborators" (show all tasks, they're all mine) from "genuinely shared team project" (filter to assigned-to-me). The collaborator count alone isn't the right signal — "At Work" has collaborators but is functionally personal. The Todoist workspace/project model may have a field that distinguishes these; worth investigating the project object's `is_shared` or `workspace_id` fields.

The workspace_id field on the Todoist Project object is the correct signal for personal vs team project distinction — not is_shared, not collaborator count. Personal projects (workspace_id=None) show all tasks even with collaborators added; workspace projects (workspace_id set) auto-filter to the current user's assigned tasks. The skill was rewritten to be user-agnostic: no hardcoded project names or team member details. The GTD methodology reference was built by synthesising multiple sources but the real quality gate was comparing against Sameer's 10-point induction doc — it exposed that the methodology was using Allen's weekly review ordering instead of Peake's "Get Current before getting Clear", and missing several framings that make GTD concepts land (types not sizes, mental RAM, pressure release valve). Synthesising is harder than combining: the rewrite was shorter despite adding content.

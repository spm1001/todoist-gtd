The CLI is transitioning from single-user to team use. The SDK (todoist-api-python v4) doesn't wrap the `/api/v1/user` endpoint, so identity is resolved via a direct httpx call to the same REST v1 base URL. This is a seam — if Todoist changes the user endpoint, the SDK changelog won't flag it.

The deeper design question right now is how to distinguish "personal project with collaborators" (show all tasks, they're mine) from "genuinely shared team project" (filter to assigned-to-me). Collaborator count alone isn't the right signal — "At Work" has collaborators but is functionally personal. The Todoist project object may have an `is_shared` or `workspace_id` field that makes this distinction; worth investigating before hardcoding heuristics.

The team is now using batterie, so generalisation is a real concern, not theoretical. The skill rewrite (tgt-nujeni) needs to be user-agnostic — no hardcoded personal setup. The current auto-filter logic treats unassigned tasks as "mine" (`assignee_id in (my_id, None)`), which is wrong for shared team projects where unassigned = untriaged. Personal area: show everything. Shared area: assigned-to-me only, with `--unassigned` for triage.

## Gotchas
- Version not bumped ‚Äî still 0.4.3 from before this session's changes. Bump to 0.4.4 next session.
- `get_current_user()` makes a fresh HTTP call every invocation ‚Äî no caching yet. Fine for single calls but would compound if called from multiple places.
- The auto-filter includes unassigned tasks as "mine" (`assignee_id in (my_id, None)`). This is wrong for shared team projects where unassigned = untriaged. See tgt-cagaci.
- The `tasks` command now fetches comments for every task unconditionally. On large lists with `--team`, this means many sequential HTTP calls. No artificial delay (the 0.2s `RATE_LIMIT_DELAY` only applies in `flatten.py`), but each is a roundtrip.

## Risks
- SDK doesn't wrap `/api/v1/user` ‚Äî our `get_current_user()` calls it directly. If Todoist changes this endpoint, the SDK changelog won't flag it.
- Comment fetching for all tasks on `--team` with a large project could hit rate limits (74+ sequential calls).
